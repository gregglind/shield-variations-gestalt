Firefox A/B Experiments Using Shield

from possible to practical

The Need

Firefox has no unified, end-to-end, system for CONTROLLED A/B experiments in release.

We can do this today, using SHIELD

  1. recruit: ask (notification bar) in release.

  2. consent: effects and data collected

  3. download: hosted somewhere safe.

  4. do: addons that have their own branch logic.

  5. record: using Unified Telemetry

  6. report: Does it make firefox Better? (live)
